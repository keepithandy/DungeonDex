DungeonDex v1.3.45 — Combat Page V2

Scope:
- Combat UI/layout/presentation pass only.
- Redesigned the combat page structure for better visual stability and mobile ergonomics.
- Replaced the v1.3.44b Combat Spam Stability Hotfix with a proper V2 layout.

What changed:

Visual/Layout:
- Redesigned combat page presentation using a flex column layout instead of forced grid row height-locking.
- Improved combat layout stability: action buttons now use position: sticky inside the combat device
  panel, keeping them anchored during rapid spam-fighting without requiring rigid screen height constraints.
- Improved mobile combat ergonomics: monster stage is always visible (no longer hidden on small phones),
  combat log scrolls internally with a sensible max-height, and buttons remain stable on tap.
- Combat log (run-log-list) now scrolls with overflow-y: auto in a clamped height region so new combat
  lines do not push the action buttons downward.
- Action button area uses a sticky + gradient fade approach so buttons stay in place visually during
  combat log growth.
- Removed ugly overcorrections from v1.3.44b: no more display:none on monster stage at small widths,
  no more 100dvh screen height locking, no more grid-template-rows rigid sizing on the combat shell.
- Elite info, HP cards, stat grid, and enemy header all retain their readable sizes and are not clipped.
- Responsive scaling via clamp() ensures proportional sizing across desktop, tablet, and iPhone viewports.

Version labels:
- BUILD and VISIBLE_VERSION_LABEL updated to DungeonDex v1.3.45.
- Cache buster strings in index.html updated to 1.3.45-combat-page-v2.
- Settings panel version label updated.

- No enemy scaling.
- No item scaling.

- No gold rewards.
