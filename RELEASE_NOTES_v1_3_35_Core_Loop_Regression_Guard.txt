DungeonDex v1.3.35 - Core Loop Regression Guard

Purpose:
This is a stability pass for the existing DungeonDex loop before 1.4 work begins. It does not add new gameplay systems, change combat balance, alter scaling, change the economy, or reorganize files.

Core fixes:
- Replaced the long visible build treatment with the quiet player-facing label: DungeonDex v1.3.35.
- Updated cache-busting strings and the service worker cache key to v1.3.35 Core Loop Regression Guard.
- Added defensive active-run recovery for incomplete run/combat states so Continue Run and save recovery return safely to Lowfire instead of leaving a stuck run.
- Hardened start-run and next-encounter setup against missing run containers, failed encounter creation, and invalid HP values.
- Hardened combat action dispatch so only attack, skill, guard, and extract can fire, and invalid active combat state recovers cleanly.
- Preserved combat math, enemy scaling, item scaling, merchant stock scaling, forge scaling, and currency tuning.
- Added click/keyboard activation support to combat buttons while keeping pointer activation for mobile controls.
- Guarded core DOM bindings and render targets so missing or stale button/panel references do not break the whole render pass.
- Hardened save/load normalization for malformed player objects, invalid HP/maxHP values, invalid active run floor 0, and null items in old currency migration paths.
- Kept extraction and defeat flows returning to town with run history and reward snapshot behavior intact.
