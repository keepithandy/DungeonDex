DungeonDex v1.4.9a — Live Server Cache / Build Label Fix

Problem:
VS Code Live Server could show stale labels like v1.4.5 or v1.4.6 because an older service worker cache could still control localhost and serve old index/script assets.

Fixes:
- Added a canonical DUNGEONDEX_BUILD value in index.html.
- Force-refreshes the visible build label/title back to DungeonDex v1.4.9a after DOMContentLoaded, load, and late startup.
- Disables service worker registration on localhost / 127.0.0.1 / 0.0.0.0.
- On local dev, unregisters old service workers and deletes DungeonDex caches.
- Keeps service worker registration enabled for deployed/itch builds.
- Updated app.js runtime pointer from v1.4.7 to v1.4.9a.
- Updated DevTools extension cache strings to use the current global build query string.

Local cleanup steps after pulling:
1. Stop Live Server.
2. Run git pull.
3. Start Live Server again.
4. Open the page.
5. Refresh twice if the first load was still controlled by an old service worker.
6. If needed, open Chrome DevTools > Application > Service Workers > Unregister, then Application > Storage > Clear site data.
