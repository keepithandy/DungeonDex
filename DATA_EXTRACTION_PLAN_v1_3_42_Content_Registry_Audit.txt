DungeonDex v1.3.42 - Content Registry Audit & Data Extraction Prep

Purpose
- Prepare for future structured data extraction without changing runtime behavior.
- This pass is audit/planning/tooling only.
- No data was moved into JSON.
- No fetch() calls, runtime JSON loading, build step, dependency, or app.js split was added.

  - RARITIES
  - rarityClass, itemRarityKey, getRarityCardClass
  - cappedRarityForLevel, weightedRarityForLevel, rarityStatMultiplier
- Item generation:
  - SLOT_ORDER, BASES, PREFIXES, SUFFIXES, TRINKET_SUFFIXES, MAKERS, THEMES
  - generateGear, expectedGearRating, lootDropChance, shouldDropLoot
- Affix and elite modifier definitions:
  - MONSTER_AFFIXES
  - ELITE_MODIFIERS and elite modifier helper functions
- Enemy generation:
  - MONSTER_FAMILIES, MONSTER_TYPES, MONSTER_SKILLS
  - generateMonster, expectedMonsterPowerAtDepth, deepMonsterPowerMultiplier, dangerRatingForDepth
- Boss generation and boss labels:
  - BOSS_INTERVAL
  - BOSS_FLOOR_NAMES
  - boss depth/name helper functions
- District and location names:
  - DISTRICT_DATA
  - district arrival, tone, and zone helper functions
- Merchant stock and district wares:
  - DISTRICT_MARKET_WARES
  - GOLD_SINK_DEFAULTS
  - buildMerchantStock, merchantGear, rollMerchant, buyMerchantItem, buyDistrictWare
- Relic and mythic set definitions:
  - MYTHIC_SET_SLOTS, MYTHIC_SET_SLOT_ALIASES, BASE_SLOT_ALIASES
  - MYTHIC_SET_DEFINITIONS
  - mythicSetDropChance, shouldDropMythicSetPiece, generateMythicSetPiece
- Vow definitions:
  - No explicit vow registry detected.
- Quest, contract, and reward tables:
  - ELITE_CONTRACTS, ELITE_CONTRACT_RISK_DEFAULTS, ELITE_CONTRACT_ID_ALIASES

  - CHARTER_EARLY_STEP, CHARTER_EARLY_LIMIT, CHARTER_LATE_STEP

- Static lore and non-runtime copy:
  - LORE_SNIPPETS
  - SESSION_LORE_LINE
  - spawnQuestLore

Safest future extraction candidates
- District display names, lines, moods, and static flavor copy from DISTRICT_DATA.
- Boss floor display names from BOSS_FLOOR_NAMES, as labels only.
- Lore snippets and static archive-style copy that is not embedded in reward/combat/save behavior.
- Rarity display labels and CSS class names, if separated from rarity multipliers and chances.
- Affix display labels, if separated from elite modifier stat multipliers, weights, reward multipliers, and combat hooks.
- Static non-runtime documentation tables.

Extract first
- A small display-only file would be the safest first pass, for example:
  - district display name/line/mood copy
  - lore snippets
  - boss floor labels
- That future pass should keep the current plain-browser launch path working and should include offline, file://, mobile, Koder, itch, and GitHub Pages checks before runtime JSON loading is considered.

Keep in app.js for now
- Item power formulas, rarity multipliers, rarity chances, loot drop chances, and generated item values.
- Enemy power, HP, guard, speed, reward, elite, boss, and deep scaling formulas.
- Elite modifier stat/reward behavior and combat hooks.
- Merchant stock generation, merchant purchases, district ware side effects, discounts, and gold sink state.
- Mythic drop chance, mythic set-piece generation, mythic slot aliasing, and set bonus behavior.
- Deep Stair Charter unlock spacing, costs, permanent charter ownership, start behavior, and death/extraction checkpoint logic.
- Quest progress banking, pending reward shape, run-history reward snapshots, and reward payout logic.
- Save-state shaping, localStorage keys, migrations, and compatibility helpers.
- Combat logic and combat math.

Do not extract yet
- Anything that changes combat math, enemy scaling, item scaling, mythic scaling, merchant behavior, Deep Stair Charter behavior, gold rewards, save format, localStorage keys, or mobile layout.
- Anything requiring fetch(), external files at runtime, npm, a build step, TypeScript, or splitting app.js.

Audit tool
- Added tools/audit_content_registries.py.
- The tool reads app.js as text and prints a readable registry map with line numbers and coupling warnings.
- It uses Python standard library only.
- It does not execute app.js, use eval, modify files, or require internet access.
