DungeonDex v1.3.37 - Mobile Dungeon UI Polish

Purpose:
This is a mobile-first presentation pass for the core dungeon and combat loop. It improves phone readability, button rhythm, combat log clarity, and light merchant/inventory scanning without changing combat math, scaling, economy, save format, or systems.

Player-facing label:
- The visible label is short: DungeonDex v1.3.37.
- No long all-caps build banner was added.
- The full pass name is used only for internal build/cache labels and these notes.

Mobile combat polish:
- Tightened the active combat shell width, padding, and panel rhythm for phone screens.
- Made floor/encounter context easier to scan with a cleaner top strip and safer text overflow.
- Improved enemy header hierarchy with larger readable names, better meta line spacing, and restrained letter spacing.
- Rebalanced enemy stage height so it still frames the encounter without wasting too much vertical space.
- Increased HP card clarity with stronger enemy/player borders, more readable HP text, and slightly stronger HP bars.
- Kept combat stats compact while improving spacing and numeric readability.

Combat log and result message polish:
- Increased the visible combat feed from 2 recent entries to 3 recent entries.
- Improved combat feed max-height, line spacing, wrapping, and first-line emphasis.
- Preserved scroll containment and iOS momentum scrolling behavior.
- Gave extraction and defeat popups slightly better line-height for mobile reading.

Button and tap-target polish:
- Refined Attack, Skill, Guard, and Extract buttons for a slimmer but still tappable 44-46px mobile target.
- Improved one-row combat action spacing so important actions remain easy to hit without feeling chunky.
- Kept Enter Dungeon, Continue Run, Rest, merchant, inventory, archive, and combat action wiring intact.
- Lightly improved mobile town action spacing and merchant/inventory item action targets.

Inventory and merchant polish:
- Lightly tightened mobile merchant, district ware, loot, and equip card padding.
- Improved small item summary line-height and action gaps on mobile.
- Kept this secondary to combat flow and avoided economic or item-scaling changes.

Preserved from v1.3.35 and v1.3.36:
- Core-loop guard work remains intact.
- Emberfall, Lowfire, Hollow Stair, and Warden identity language remains intact.
- No faction, vow, roadmap, or 1.4 feature work was added.
