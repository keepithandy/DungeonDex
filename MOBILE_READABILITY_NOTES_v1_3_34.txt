DungeonDex v1.3.34 - Mobile Readability Final

1. Areas reviewed
- Lowfire District header, dungeon entry actions, Rest cost chip, Deep Stair Charters.
- Lowfire Market, merchant stock cards, District Wares, Relic Forge, Contract Board, Elite Contract Board.
- Gear, inventory cards, filters, Dex, Archive, Save/Reset controls.
- Run screen idle state, active combat CSS, combat feed, HP rows, combat action buttons, extraction/defeat popup styles.
- Intro threshold modal and mobile viewport/scroll behavior.

2. UI adjustments made
- Added a final v1.3.34 CSS override block instead of rewriting older layout sections.
- Improved mobile town hierarchy with a single-column district panel and cleaner Enter Dungeon, Rest, and cost-chip stacking.
- Gave charter buttons a safer mobile grid, with a one-column fallback on very narrow screens.
- Tightened but clarified merchant, district ware, forge, objective, elite contract, equipment, and inventory card rhythm.
- Improved small text line-height, overflow wrapping, tag row gaps, and long-name handling.
- Made the forge action full width inside its panel for clearer tap targeting.
- Improved intro modal viewport fit, scroll containment, and threshold metadata spacing.
- Kept combat compact while slightly improving action button height, feed line-height, HP row spacing, and log scroll comfort.
- Updated internal build/cache labels to the v1.3.34 mobile readability line.

3. Mobile-specific risks found
- The app still depends on the existing app-shell scroll container for Itch/mobile iframe safety; this was preserved.
- Active combat has very dense small-screen rules below 520px and 370px. The final pass adjusted readability conservatively without changing the combat shell structure.
- Generated item names, reward values, and currency strings can still be long. The pass improves wrapping, but extreme save states should be checked during regression.
- The first-load intro modal can still occupy most of a short phone viewport; scrolling is preserved and buttons stay in the modal flow.

4. CSS areas intentionally left untouched
- Core html/body/app-shell iframe scroll architecture.
- Combat stage proportions, monster visuals, and active combat shell layout.
- Desktop grid structure and existing panel hierarchy.
- Existing rarity palette, district theme surfaces, and card identity styling.
- Sticky navigation architecture and service worker behavior beyond cache-label updates.

5. Remaining readability concerns
- Real iOS Safari and embedded Itch browser testing is still needed.
- Ultra-narrow screens can still make combat labels and currency-heavy buttons tight.
- Active combat was checked through render markup and CSS review, but not through a live click/screenshot session.
- Old historical note files still contain prior audit references; runtime files were the focus for stale-label cleanup.

6. Recommended next pass
- DungeonDex v1.3.35 - Save / Enter Dungeon / Combat Regression.
