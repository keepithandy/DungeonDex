DungeonDex v1.3.44 - Reward Reveal & Run Summary Polish

Summary
- Player-facing reward and run-result presentation polish only.
- Improved reward reveal wording for room rewards, loot finds, no-gear results, elite spoils, boss spoils, mythic finds, and milestone rewards.
- Improved floor clear, extraction, defeat, latest-run, and run history wording so the player can quickly tell what was gained, banked, lost, or carried forward.
- Clarified reward source labels where the existing combat feed and popup structure already supported them: Boss Spoils, Elite Spoils, Room Reward, Milestone Reward, Extraction Haul, and Mythic Find.
- No gameplay changes.
- No save changes.
- No balance changes.

Reward and result clarity
- Normal victory rewards now read as Room Reward instead of a generic unsecured result.
- Elite victories now read as Elite Spoils, including the existing elite risk bonus roll messaging.
- Boss victories and bounty relics now read as Boss Spoils.
- Mythic loot reveals now call out Mythic Find while preserving the same item generation and drop paths.
- No-drop normal encounters now say that no gear was found while the player keeps moving with the existing coin result.

Run summary clarity
- Extraction messaging now uses Extraction Haul and states that Lowfire banks the haul and marks the run complete.
- Defeat messaging now uses Run Lost / the run ends here language and states that unsecured rewards were lost while banked gear and wallet stayed safe.
- Run history fallback wording now better distinguishes secured extractions from lost runs.
- Floor and room milestone reward lines now identify the source before listing the same unsecured reward amounts.

Version/cache
- Advanced internal build/cache labels to v1.3.44.
- The visible label remains short: DungeonDex v1.3.44.
