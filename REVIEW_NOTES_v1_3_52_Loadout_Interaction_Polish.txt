DungeonDex v1.3.52 - Loadout Interaction Polish Review Notes

Scope checked
- Gear screen loadout renderer and inventory card markup.
- Loadout-specific CSS in the existing loadout polish block.
- Visible version labels, runtime build constants, asset query strings, and service worker cache key.
- Existing equip, sell, sell-junk, sell-all, and Enter Dungeon binding paths.

Findings and fixes
- Equipped and inventory gear were still visually close because both inherited the same rarity-card treatment. Equipped cards now get a warmer border, a left ember accent, and a slightly stronger worn-gear surface while inventory cards stay quieter.
- Gear card hierarchy now reads name first, then slot/type plus rarity, then level, then Power and the key stat. Inventory actions remain at the bottom of each card.
- Empty equipment cards no longer repeat both Empty and No item copy. They now keep the slot name and Empty state only.
- Loadout and inventory helper copy was shortened to reduce repeated wording.
- Mobile loadout spacing was tightened with smaller panel gaps, while inventory action buttons now keep larger tap targets on phone-width screens.
- The visible player-facing label remains short: DungeonDex v1.3.52.

No gameplay or persistence changes
- Combat math unchanged.
- Loot generation unchanged.
- Economy values unchanged.
- Dungeon progression unchanged.
- Merchant inventory generation unchanged.
- Item rarity logic unchanged.
- Save structure and storage key unchanged.

Residual notes
- The pass intentionally stays inside the current loadout surface. It does not add a new unequip system or change the existing equip/swap behavior.
