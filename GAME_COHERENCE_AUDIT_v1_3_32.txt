DungeonDex v1.3.32 - Game Coherence Audit

1. Executive Summary

Current condition: Needs polish, not a systems rewrite.

DungeonDex has a recognizable core loop:
Lowfire District -> Enter Dungeon -> Combat -> Loot -> Extract / Die -> Merchant / Forge / Archive -> Repeat.

The loop is playable and most visible systems now point back to dungeon runs. The largest coherence issue is not missing functionality; it is naming and presentation drift from several incremental passes. Town mostly reads as a hub now, but the first-load intro modal still repeats dungeon-entry framing outside the town panel, Warden Objectives still leak "quest" terminology in logs/history, and several debug/audit helpers and old patch comments remain in the runtime files.

No new v1.4 labels were introduced in runtime files. No Primary Descent text or slug remains in runtime files. No zip/package was created during this audit.

2. Release Blockers

No hard gameplay blockers found for a v1.4 coherence release.

Potential public-coherence blocker if left untouched:
- First-load intro modal: It presents "Dungeon Threshold" with its own Enter Dungeon button before the player engages with the Lowfire District hub. It is functional, but it can feel like a second dungeon-entry surface. This should be aligned, simplified, or made less prominent before public v1.4 if the goal is one clear town hub.

3. High-Priority Cleanup

1. Align or reduce the first-load intro modal.
   - Current: introProgressMarkup('modal') renders "Dungeon Threshold", Return Point, and an Enter Dungeon button.
   - Risk: Feels like a second entry area after the Primary Descent cleanup.
   - Recommendation: In v1.3.33, keep the modal if needed, but make it clearly an onboarding/status overlay rather than another primary town panel.
   - Rating: Needs fix before v1.4.

2. Clean up quest/objective naming leaks.
   - Current visible UI says Warden Objectives, but app logs/history can still say "Quest complete" and "quest progress".
   - Risk: The player sees a mixed identity: objectives vs quests vs contracts.
   - Recommendation: Rename visible player-facing "Quest complete" and run-history "quest progress" copy to "Objective complete" / "objective progress". Do not change save keys yet.
   - Rating: Needs fix before v1.4.

3. Remove or hide vow-language that implies a future system.
   - Current: item suffix "of Vows" and lore line "unfinished vows".
   - Risk: The user has explicitly deferred the real v1.5 Vow system; visible vow language can imply a system that does not exist.
   - Recommendation: Replace these two flavor strings with neutral Emberfall terms in a small copy-only pass.
   - Rating: Needs polish before v1.4.

4. Decide whether Deep Stair Charters should remain inside the Lowfire District banner.
   - Current: Charters are functionally attached to the dungeon-entry hub, which is coherent, but the copy is long and pushes the top town card taller.
   - Risk: Mobile first screen can feel dense before the player reaches market/forge.
   - Recommendation: Keep the placement for now, shorten copy and button labels only.
   - Rating: Needs polish.

5. Runtime build/cache metadata remains v1.3.31b.
   - Current: BUILD, index.html query strings, and sw.js cache are v1.3.31b.
   - Risk: Fine for this audit because runtime code was not changed, but update deliberately in the next implementation pass if public artifacts are produced.
   - Recommendation: Do not rename to v1.4 yet. If v1.3.32 becomes public, update metadata in a dedicated version-only pass.
   - Rating: Needs fix only if shipping this exact build.

4. Medium-Priority Polish

1. Lowfire Market density.
   - Merchant stock, District Wares, active sink pills, and refresh are coherent, but the panel can become visually heavy with four stock cards plus wares.
   - Recommendation: Keep systems in place; polish card density and section rhythm later.
   - Rating: Needs polish.

2. District Wares identity.
   - Current: Useful gold sinks, but "District Wares" and "coin sinks" are still mechanical phrasing.
   - Recommendation: Keep the name; reduce copy around "gold sinks" in public UI. Use effect-first text.
   - Rating: Needs polish.

3. Dex naming.
   - Current: Tab says Dex, panel says Emberfall Index, sections say Monster Register and Relic Register.
   - Risk: Not broken, but the hierarchy is not obvious.
   - Recommendation: Pick one naming family: Dex / Register / Index. "Dex" is concise for tab; "Emberfall Index" is acceptable if copy explains less.
   - Rating: Needs polish.

4. Gear / inventory density.
   - Current: Gear screen has loadout, filters, inventory, rarity tags, stat pills, sell buttons.
   - Risk: Dense on mobile, but useful.
   - Recommendation: Avoid structural changes; only tighten typography and wrapping after screenshot review.
   - Rating: Needs polish.

5. Archive/System Notes.
   - Current: Archive includes Run History, Archive Log, and System Notes.
   - Risk: "System Notes" is useful but feels development-oriented beside narrative history.
   - Recommendation: Rename to a more in-world but still clear label, or move it lower. No behavior change needed.
   - Rating: Needs polish.

6. Boss naming and depth language.
   - Current: Variables and some copy mix floor/depth/room/chapter.
   - Risk: Player sees Depth/Floor/Room/Chapter in several places. Mostly understandable, but release copy should choose a hierarchy.
   - Recommendation: Use "Depth" for raw progress and "Floor / Room / Chapter" as detail. Avoid "floor" when referring to unlock depth in visible copy.
   - Rating: Needs polish.

5. Defer to v1.5+

Do not add these before v1.4:
- Real Vow system.
- Factions.
- Reputation.
- Town evolution.
- New dungeon branches.
- New crafting systems beyond current Relic Forge.
- New economy layers.
- Additional contract types beyond the existing Elite Contract Board.
- Merchant personalities or faction stock.

6. Town Coherence Notes

Lowfire District hub:
- Purpose: main staging hub, enter/continue, rest, rest cost, charter context.
- Current usefulness: high.
- Loop connection: direct start/restart point for dungeon loop.
- Status: Needs polish, not redesign.
- Notes: The Lowfire banner now owns the single in-town entry path. This is coherent. Charters inside the banner make sense, but copy is long.

Rest:
- Purpose: restore HP for a visible cost.
- Current usefulness: high.
- Loop connection: recovery before next run.
- Status: Ready.
- Notes: Clear enough. Keep cost visible.

Lowfire Market:
- Purpose: buy gear and run-support wares.
- Current usefulness: high.
- Loop connection: spend run rewards to improve next run.
- Status: Needs polish.
- Notes: "Merchant stock, district wares, and coin sinks" is accurate but slightly mechanical. Consider shorter, effect-first copy.

District Wares:
- Purpose: permanent/limited gold sinks unlocked by progress.
- Current usefulness: medium/high.
- Loop connection: spends gold earned from runs on next-run bonuses or protection.
- Status: Needs polish.
- Notes: Useful, but "gold sink" language reads like design terminology. Avoid adding more economy layers.

Relic Forge:
- Purpose: convert sparks/shards into a rare+ item.
- Current usefulness: high once resources exist.
- Loop connection: rewards from dungeon feed forge, forge feeds gear.
- Status: Ready with minor copy polish.
- Notes: Keep current behavior. Do not expand crafting.

Deep Stair Charters:
- Purpose: buy/activate permanent deeper starts after safe extraction milestones.
- Current usefulness: high for long runs, low early.
- Loop connection: progression convenience tied to extraction safety.
- Status: Needs polish.
- Notes: Functional and relevant, but the "Every 40 safe depths... mega-depth..." copy is long for a top-town card.

Contract Board / Warden Objectives:
- Purpose: objectives and elite risk jobs.
- Current usefulness: medium/high.
- Loop connection: rewards players for kills, loot, equipment, and elite fights.
- Status: Needs fix before v1.4.
- Notes: UI name is better than Vows. Player-facing "quest" logs/history still need copy cleanup.

Gear:
- Purpose: equip loot, sell extras, filter inventory.
- Current usefulness: high.
- Loop connection: loot -> power -> deeper runs.
- Status: Needs polish.

Dex:
- Purpose: record discovered monsters/relics.
- Current usefulness: medium.
- Loop connection: supports collection fantasy, not core power.
- Status: Needs polish.
- Notes: Keep, but consider making its empties and names less generic.

Archive:
- Purpose: run history, losses, rewards, archive log, system notes.
- Current usefulness: high for clarity.
- Loop connection: shows extraction/death outcomes.
- Status: Needs polish.

Save / Reset:
- Purpose: persistence and full reset.
- Current usefulness: high.
- Loop connection: infrastructure, not gameplay.
- Status: Ready.

First-load intro modal:
- Purpose: orient player to current return point and start/continue.
- Current usefulness: medium.
- Loop connection: direct entry, but duplicates Town entry.
- Status: Needs fix before v1.4.
- Notes: Keep if it improves onboarding; otherwise reduce to a dismissible status overlay.

9. Mobile Readability Risks

Needs manual screenshot pass:
- Lowfire District hub plus Deep Stair Charter copy may be tall on 360px-wide phones.
- Merchant stock cards are dense, especially with money markup and item summaries.
- District Wares cards combine rarity, cost, effect, summary, status, and buy reason.
- Elite Contract Board card has objective/risk/reward/action; reward money can wrap.
- Gear cards have rarity, name, ilvl, slot/maker, stat pills, set previews, equip/sell controls.
- Archive run-history cards include many metadata pills and reward summaries.
- Intro modal uses a rich threshold layout and can compete with the first screen on small itch/browser frames.
- Fixed bottom tabs require continued padding checks to avoid content being hidden near the bottom.
- Combat action buttons are compact but likely acceptable; keep screenshot checks on 360px and 390px widths.
